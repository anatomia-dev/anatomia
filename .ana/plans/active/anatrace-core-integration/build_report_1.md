## Implementation Decisions

- **Single byte-read for derive + hash.** `captureProvenanceAtSave` reads the transcript bytes once and computes both the `transcript_hash` and the derived counts from those exact bytes (rather than calling `deriveTranscript` and re-reading for the hash). This guarantees the hash attests precisely the bytes that were derived — the spec's "same bytes" requirement — and avoids a second filesystem read. `deriveTranscript(path, harness)` is kept as the public seam (used by the derive unit tests) and delegates to the same internal `deriveCountsFromBytes`.
- **`transcript_hash` ordered before `derived`** in the written object, immediately after `captured_at`, matching the wrapper-metadata grouping in the spec mockup.
- **Claude fixture enrichment over literal change.** Rather than re-baseline the Claude token literals downward, I added the `message.id` and `tool_use_id` fields that every real Claude transcript carries; with them, core reproduces the prior derive's exact counts (1500/800/150/300, tests 12/2, files 1, duration 21000). Documented as a fixture-realism fix, not a literal regression.
- **Codex command tool = `exec_command`.** Core recognizes `exec_command` (not the old fixture's `shell`) as a command tool, so the rewritten Codex fixture uses it to keep a meaningful `commands_run === 1` and test-count assertion.

## Deviations from Contract

### Re-baselined literals (AC12 requires justification)

- **Codex `tokens.input`: 300 → 220.** Core's Codex adapter reports FRESH input — gross `input_tokens` (300) minus `cached_input_tokens` (80) = 220 — whereas the old hand-derive recorded the gross total. Asserted with an inline note in `forensics-derive.test.ts`.
- **Codex `duration_ms`: 30000 → 28000.** Core measures the span across the timestamped events it folds (first usage event `00:00:02` → last `00:00:30` = 28000ms), not the raw first/last transcript line. Documented inline.
- **Claude/Codex `derived` gains `derive_version: "3"`** (new core field) — additive, not a changed value.

These are genuine re-baselines against core's documented behavior, not silent snapshot regeneration. All other Claude counts are unchanged (fixture enriched to carry the ids real transcripts have — see Implementation Decisions).

### Contract assertions

None — every Phase-1 assertion (A001–A012) is satisfied as specified. A002 is satisfied by deletion + the contract's mechanical `not_contains` source matcher rather than a dedicated `@ana` test (noted in Open Issues); all others carry `@ana` tags.

- **`proof-card-golden.test.ts` `toMatch(/TOTAL.*table v3/)` → `/TOTAL.*table 2026-06-08/`.** The old assertion encoded the pre-AC6 behavior (display = per-record stamp). AC6 sources the displayed version from the `CostResult`, which is the real `2026-06-08`. This is the assertion AC6 inverts.
- **Golden snapshots regenerated.** Sole content change is the TOTAL footer version label (`v3` → `2026-06-08`); all costs are byte-identical (`$4.02`, `$6.68`, `$1.77`, `$1.07`), proving AC6 changes no displayed cost.

User-approved approach: leave rendering untouched (spec scope), adjust fixtures. The `≤ 80` guarantee is kept intact and meaningful. See Open Issues for the underlying layout overflow.
